## 🚀 How to Get Started (Step by Step)

### Step 1: Read the Core (This Week)
1. Read **AGENTS.md** — understand the startup sequence
2. Read **SOUL.md** — customize with your personality
3. Skim **OPERATING_RULES.md** — understand the pre-flight checklist
4. Run **FIRST-RUN.md** — have the conversation with your agent

### Step 2: Populate Your Memory (First Month)
1. Create **MEMORY.md** with your businesses, goals, and how you work
2. Have your agent read the **wiki/** files relevant to your business
3. Start saving daily notes in **memory/YYYY-MM-DD.md**

### Step 3: Load Your Tools (First Week)
1. Read **TOOLS.md** to see what tools you need
2. Create **credentials.env** with your actual API keys
3. Your agent can now access your systems

### Step 4: Leverage Frameworks (Ongoing)
1. When you communicate externally, reference **wiki/communication-playbook.md**
2. When you face a problem, search **wiki/frameworks/**
3. When you need a tool, check **solutions-playbook/tools-master-database.md**

## 🎯 Key Concepts You'll See

### **Tier 1/2/3 Memory Architecture**
- **Tier 1:** MEMORY.md — active flags, rules, pointers (small, always loaded)
- **Tier 2:** wiki/ folder — domain knowledge (read when relevant)
- **Tier 3:** Project STATUS.md files — deep context (read only for that project)

### **Pre-Flight Checklist**
Before asking for help or flagging something as blocked, your agent checks:
1. Do we have this credential?
2. Do we have this tool?
3. Has this already been built?
4. Has this already been decided?
5. Did we do this recently?

### **Frameworks + Depth**
Your agent doesn't just answer questions — it applies frameworks. When communicating, it uses 10 proven frameworks. When you face a problem, it searches the framework database.

### **Preset Sync Rule**
Every time something new is learned/built, the preset system is updated. This ensures that what you learn benefits future instances of your agent + any other agents you create.

---

## 🔄 How Updates Work

When you discover a new framework, tool, or process:
1. **Your agent updates** its own files (AGENTS.md, OPERATING_RULES.md, etc.)
2. **Simultaneously** updates these preset files
3. **Documents** the change in PRESET_SYNC_LOG.md
4. **Verifies** with fresh eyes that the change is actually there
5. Future agents get the update automatically

This is how the system gets smarter over time — everything you learn transfers to the next agent.
